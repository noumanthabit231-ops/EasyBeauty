# BeautyTap — SaaS-витрина для магазинов косметики

Аналог Taplink для косметических магазинов. Владелец оформляет подписку, создаёт
страницу магазина, добавляет товары/категории/акции. Клиенты собирают корзину и
оформляют заказ через **WhatsApp** (без эквайринга).

## Стек
- **Next.js 14** (App Router) + TypeScript + Tailwind CSS
- **Supabase** — база данных, Auth, Storage, RLS
- **Vercel** — хостинг

## Возможности
- Регистрация владельцев, у каждого свой кабинет
- Редактируемо: название, адрес, город, номер WhatsApp, валюта, цвет кнопок, логотип, обложка
- Категории, товары (фото, цена, старая цена, бейджи), акции и промокоды
- Публичная витрина по адресу `site.com/{slug}` с корзиной → заказ в WhatsApp
- Супер-админ панель `/admin` — ручное управление подписками магазинов

## Установка

### 1. Настроить Supabase
1. Создайте проект на [supabase.com](https://supabase.com).
2. Откройте **SQL Editor** → вставьте и выполните весь файл [`supabase/schema.sql`](supabase/schema.sql).
   Это создаст таблицы, RLS-политики, Storage-бакет `store-media` и триггеры.
3. В **Authentication → Providers → Email** можно выключить *Confirm email*,
   чтобы регистрация сразу входила в кабинет (удобно для старта).

### 2. Переменные окружения
Скопируйте `.env.local.example` → `.env.local` и заполните из **Project Settings → API**:

```
NEXT_PUBLIC_SUPABASE_URL=...
NEXT_PUBLIC_SUPABASE_ANON_KEY=...
SUPABASE_SERVICE_ROLE_KEY=...        # секрет! только сервер
NEXT_PUBLIC_SUPER_ADMIN_EMAIL=...
```

### 3. Запуск локально
```bash
npm install
npm run dev
```
Откройте http://localhost:3000

### 4. Назначить себя супер-админом
1. Зарегистрируйтесь на `/register` под своим email.
2. В Supabase → SQL Editor выполните:
   ```sql
   update public.profiles set role = 'super_admin'
   where email = 'ваш-email@example.com';
   ```
3. Теперь вам доступна панель `/admin`.

## Деплой на Vercel
1. Залейте папку `cosmetics-saas` в отдельный git-репозиторий.
2. На [vercel.com](https://vercel.com) → **New Project** → импортируйте репозиторий.
3. В **Settings → Environment Variables** добавьте те же 4 переменные из `.env.local`.
4. Deploy. Магазины будут доступны по `https://ваш-домен.vercel.app/{slug}`.

## Как это работает
- **Подписка** активируется вручную вами в `/admin` (кнопки +1 мес / +3 мес / +1 год).
  Пока подписка неактивна — витрина магазина скрыта от клиентов.
- **Заказ** формируется на стороне клиента и открывает `wa.me` с готовым текстом.
  Деньги проходят напрямую между магазином и покупателем — платформа платежи не обрабатывает.

## Структура
```
src/
  app/
    page.tsx              лендинг
    login, register       авторизация
    dashboard/            кабинет владельца (обзор, товары, категории, акции, настройки)
    admin/                супер-админ панель
    [store]/              публичная витрина магазина
  components/             UI-компоненты (витрина, менеджеры, формы)
  lib/                    supabase-клиенты, типы, auth-хелперы
supabase/schema.sql       схема БД + RLS
```
